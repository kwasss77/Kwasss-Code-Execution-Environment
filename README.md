## 1.2 📢 Команды (пока мало)
1. math
2. print
4. exit

### 1.2.1 math
math - строго маленькими буквами. Это оператор, который решает примеры со знаками +, -, * (умножить), / (разделить). Внимание! Он поддерживает только однозначные целые числа. Он не засчитает -2, 2.5, 27 и другие. А только 0, 1, 2, 3... 9 . Пример синтаксиса: строго "m 1+3" или "m 7-3". Команду писать без кавычек. Обратите внимание что есть только команда m и ставится только один пробел перед примером. С большой буквы не зарегает. Результат выводится в консольное окно Windows (если запускать командой оттуда же).

### 1.2.2 print
print - команда которая выводит встроенные в язык переменные (ниже) или свой текст. Примеры команды (без одинарных кавычек): 'print "Привет, мир!"'; 'print [math_title]'. Итоговый текст выводится в стандартное msg окно от Windows. Оно не поддерживает русский язык. Изменено: можно писать текст в двойных кавычках (тогда внутри можно будет ставить одинарные), и наоборот. Также их вообще можно опустить (если вы не будете использовать кавычки в самом тексте).

### 1.2.3 exit
exit - просто 4 буквы. Вы должны поставить эту команду в конец .kc2e файла во избежание ошибок и вылетов, и прочих багов.


## 1.3 Как этим пользоваться??? (со стороны кодера/программиста)

### 1.3.1 🎫 Начало
Например, вы создали текстовый файл, переименовали его в test.kc2e, и написали простенькую программу:

```text
print privet
m 1+4
exit
```

Понятно да, пишет текст, считает, выходит. Дальше, вам потребуется мой язык собственно говоря (файлом), ассемблер (программа-компилятор из языка "ассемблер" в машинный код) и линкер (сборщик в .exe). Для этих целей давайте возьмём Nasm и GoLink.

### 1.3.2 🎨 Линкер + ассемблер
Скачивайте в releases версию помеченную как stable (стабильная), распакуйте архив (там два файла) туда, куда вам удобно (например в папку C:\nasm\) дальше скачайте два этих инструмента:
Nasm - https://nasm.us (для windows x64, версия 3.01)
GoLink - https://godevtool.com (для x64, скачивайте GoLink 32/64-bit linker, ссылка на этот файл ниже)
файл: https://godevtool.com Дальше, как только вы скачали, перекидывайте из архивов файлы GoLink.exe, nasm.exe и ndisasm.exe в вашу папку (в примере C:\nasm\).

### 1.3.3 📂 Итоги (без путаницы):
Итоговый вид:
```text
C: -
    | - Program files
    | - Program files (x86)
    | - nasm -
            | build.bat *для разработчиков
            | GoLink.exe
            | kc2e.asm *для разработчиков
            | kc2e.exe
            | nasm.exe
            | ndisasm.exe
    | - Windows
    | - Пользователи
```
Есть два способа запустить:
1. Интерпретатор.
2. нету, не хочется писать свой компилятор (можете за меня: bombsearch@gmail.com)

### 1.3.4 🚗 СПОСОБ ЗАПУСКА .kc2e ФАЙЛОВ
Итак, можно либо:
1. Перетащить мышкой файл .kc2e на kc2e.exe. Тогда ничего не будет видно (т.к. я не добавил никакого GUI в свой язык)
2. Запустить в консоли по текущему пути (у нас C:\nasm\) файл kc2e.exe с аргументом, которым будет ваш .kc2e файл, например: `kc2e.exe test.kc2e`. В таком случае все будет отображаться в консоли.


## 1.4 🥔 Исходный код (со стороны разработчика)
Если вы хотите взять как референс, или доработать, или исправить что-л в моём языке, то в исходном коде есть два важных файла: kc2e.asm и build.bat. Вам нужно сделать так же как в 🎨1.3.2, и тогда вы сможете делать следующие вещи:
1. С помощью npp (или других текстовых редакторов) открывать .asm
2. При открытии kc2e.asm как раз будет исходный код, вы можете его поменять
3. После ваших изменений вы можете запустить build.bat. Он соберёт из .asm файла .exe файл, годящийся для запуска своих .kc2e файлов способом выше (🚗1.3.4). Также обратите внимание, что файлы build.bat и kc2e.asm должны лежать в одной папке.
